## 📋 这个系统做什么

**自动记录**:
1. ❌ 错误和失败 (ERRORS.md)
2. ✅ 学到的经验 (LEARNINGS.md)
3. 💡 功能需求 (FEATURE_REQUESTS.md)

**自动优化**:
1. 分析失败原因
2. 总结成功经验
3. 调整内容策略
4. 避免重复错误

---

## 🎯 使用场景

**记录错误**:
```
当内容被限流/没人看时:
- 记录到 ERRORS.md
- 分析限流原因
- 避免下次再犯
```

**记录经验**:
```
当某篇内容爆了:
- 记录到 LEARNINGS.md
- 分析爆款元素
- 复制成功模式
```

**记录需求**:
```
当需要新功能时:
- 记录到 FEATURE_REQUESTS.md
- 评估优先级
- 开发实现
```

## 📝 如何录入数据

### 发布后录入

**格式**:
```markdown
## 2026-03-13 - [内容标题]

**平台**: 小红书/头条
**类型**: 情绪共鸣/干货/场景
**数据**:
- 阅读：xxx
- 点赞：xxx
- 评论：xxx
- 分享：xxx

**表现**: 好/一般/差
**分析**: 为什么好/为什么差
**改进**: 下次怎么做
```

**录入位置**:
- 表现好 → `LEARNINGS.md`
- 表现差 → `ERRORS.md`
- 新需求 → `FEATURE_REQUESTS.md`
