## 通信协议

### 直接通信 vs 总控中转

| 场景 | 通信方式 | 示例 |
|------|----------|------|
| **任务分配** | 总控→Agent | 总控分配创作任务 |
| **状态汇报** | Agent→总控 | 任务完成汇报 |
| **同级协作** | Agent↔Agent | 竞品监控师→创意注入师 |
| **紧急告警** | 任意→所有 | 质量风险告警 |

### 消息格式

```json
{
  "type": "task|report|alert|request",
  "from": "agent-id",
  "to": "agent-id|all",
  "priority": "normal|high|urgent",
  "content": {
    "subject": "消息主题",
    "body": "详细内容",
    "action": "需要的行动",
    "deadline": "截止时间 (可选)"
  },
  "timestamp": "ISO-8601"
}
```

### 响应时效

| 优先级 | 响应时间 | 示例 |
|--------|----------|------|
| **normal** | <30 分钟 | 常规任务分配 |
| **high** | <10 分钟 | 爆款分析请求 |
| **urgent** | <5 分钟 | 质量风险告警 |

## 事件触发规则

### 1. 爆款检测事件

```yaml
触发条件:
  - 竞品内容发布 24h 内
  - 阅读量>10 万 或 互动率>5%

触发行动:
  - 竞品监控师 → 创意注入师：分析可借鉴元素
  - 竞品监控师 → 头条创作：生成选题建议
  - 竞品监控师 → 总控调度官：备案

时效要求:
  - 检测后 15 分钟内完成通知
```

### 2. 质量风险事件

```yaml
触发条件:
  - 违禁词检测命中
  - 数据无来源
  - 敏感内容识别

触发行动:
  - 质量审核师 → 创作 Agent：修改建议
  - 质量审核师 → 总控调度官：风险备案
  - 如高风险：→ 禁止发布

时效要求:
  - 发现后 5 分钟内通知
```

### 3. 数据异常事件

```yaml
触发条件:
  - 阅读量低于基线 50%
  - 互动率低于基线 50%
  - 粉丝单日流失>100

触发行动:
  - 数据分析师 → 创作 Agent：数据反馈
  - 数据分析师 → 总控调度官：异常报告
  - 数据分析师 → 竞品监控师：竞品对比

时效要求:
  - 发现后 30 分钟内分析完成
```

### 4. 选题枯竭事件

```yaml
触发条件:
  - 选题池<3 个可用选题
  - 距离发布时间<4 小时

触发行动:
  - 总控调度官 → 竞品监控师：紧急选题挖掘
  - 总控调度官 → 创意注入师：紧急创意生成
  - 总控调度官 → 老麻：人工决策请求

时效要求:
  - 立即响应
```

## 强制审核流程

### 发布前审核 (所有平台)

```
┌─────────────────────────────────────────────────────────┐
│              发布前强制审核流程                          │
├─────────────────────────────────────────────────────────┤
│  1. 内容创作完成                                        │
│         ↓                                               │
│  2. 自动触发质量审核                                    │
│         ↓                                               │
│  3. 质量审核师分析 (10 分钟内)                           │
│         ↓                                               │
│  4. 审核结论                                            │
│         ├─→ ✅ 通过 → 允许发布                          │
│         ├─→ ⚠️ 修改后发布 → 创作 Agent 修改 → 重新审核  │
│         └─→ ❌ 禁止发布 → 总控决策 → 重写或放弃         │
│                                                         │
│  5. 审核记录归档 (memory/audit-logs/)                   │
└─────────────────────────────────────────────────────────┘
```

### 审核豁免

| 情况 | 是否豁免 | 说明 |
|------|----------|------|
| 紧急热点 (<1h 窗口) | ❌ 不豁免 | 可走快速通道 (5 分钟) |
| 老麻明确要求 | ❌ 不豁免 | 但可先发布后补审 |
| 纯数据报告 | ✅ 豁免 | 无创作内容 |
| 内部测试 | ✅ 豁免 | 不公开发布 |

## 会议机制

### 每日站会 (08:30, 15 分钟)

**参与者**: 总控 + 头条 + 小红书 + 数据分析 + 竞品监控

**议程**:
```
1. 昨日复盘 (数据分析师，5min)
   - 发布内容数据
   - 关键发现
   - 异常说明

2. 今日选题 (竞品监控师，3min)
   - 推荐选题 Top 3
   - 热点提醒

3. 风险提醒 (质量审核师，2min)
   - 违禁词更新
   - 平台规则变化

4. 创意分享 (创意注入师，3min)
   - 新梗/新话术
   - 可借鉴案例

5. 任务确认 (总控，2min)
   - 今日发布计划
   - 责任分配
```

### 每周评审 (周一 10:00, 60 分钟)

**参与者**: 所有 Agent + 老麻

**议程**:
```
1. 周报汇总 (数据分析师，10min)
   - 本周核心指标
   - 目标达成情况

2. 案例复盘 (轮流，20min)
   - 成功案例 1 个
   - 失败案例 1 个

3. KPI 回顾 (总控，10min)
   - 各 Agent 表现
   - 改进建议

4. 问题讨论 (开放，15min)
   - 工具需求
   - 协作问题
   - 资源申请

5. 优化决策 (老麻，5min)
   - 方向调整
   - 优先级确认
```

### 每月规划 (每月 1 号，90 分钟)

**参与者**: 所有 Agent + 老麻

**议程**:
```
1. 月度复盘 (30min)
   - 目标达成
   - 关键学习

2. 下月规划 (30min)
   - 目标设定
   - 重点项目

3. 能力建设 (20min)
   - 新技能学习
   - 工具升级

4. 团队优化 (10min)
   - 流程改进
   - 协作优化
```

## 共享状态管理

### 共享状态文件

**位置**: `shared-state.json`

**结构**:
```json
{
  "updatedAt": "ISO-8601",
  "topicPool": {
    "active": [],
    "reserved": [],
    "completed": []
  },
  "publishPlan": {
    "toutiao": {
      "scheduled": [],
      "published": []
    },
    "xiaohongshu": {
      "scheduled": [],
      "published": []
    }
  },
  "riskList": [],
  "creativeLibrary": {
    "memes": [],
    "scripts": [],
    "templates": []
  },
  "agentStatus": {}
}
```

### 状态更新规则

| 状态项 | 更新者 | 更新频率 |
|--------|--------|----------|
| topicPool | 竞品监控师 + 创意注入师 | 实时 |
| publishPlan | 创作 Agent | 实时 |
| riskList | 质量审核师 | 实时 |
| creativeLibrary | 创意注入师 | 每日 |
| agentStatus | 各 Agent | 每小时 |

### 状态读取权限

| Agent | 读取权限 | 写入权限 |
|-------|----------|----------|
| 总控 | 全部 | 全部 |
| 头条创作 | topicPool, publishPlan, creativeLibrary | publishPlan |
| 小红书创作 | topicPool, publishPlan, creativeLibrary | publishPlan |
| 数据分析师 | 全部 (只读) | agentStatus |
| 竞品监控师 | topicPool, publishPlan | topicPool |
| 质量审核师 | publishPlan, riskList | riskList |
| 创意注入师 | topicPool, creativeLibrary | creativeLibrary |

## 冲突解决机制

### 优先级规则

```
老麻决策 > 总控调度 > Agent 协商 > 默认规则
```

### 常见冲突场景

| 冲突 | 解决方式 |
|------|----------|
| 选题冲突 (多 Agent 选同一题) | 总控协调，按优先级分配 |
| 发布时间冲突 | 总控调整，避免撞车 |
| 审核分歧 (创作 vs 审核) | 总控仲裁，或老麻决策 |
| 资源竞争 (工具/数据) | 优先级高的优先 |
