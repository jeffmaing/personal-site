# HEARTBEAT.md - 总控心跳

## 安全监控模式 (2026-03-13 激活)

### 每次心跳检查

1. **exec 审批队列** - 有待审批的命令？
2. **cron 执行状态** - 有失败的任务？
3. **后台服务状态** - MCP/Dashboard 正常运行？
4. **技能加载日志** - 有异常加载尝试？
5. **发布状态检查** - 定时发布的任务是否成功？
6. **agent 存活状态** - 头条/小红书/日报 agent 是否正常？

### 告警条件

#### 🔴 紧急告警（立即通知）
- exec 审批超过 5 个未处理
- cron 连续失败 3 次
- 检测到技能注入尝试
- 发布失败（scheduled 时间过后 10 分钟仍未发布）
- agent 超时未响应（>30 分钟无活动）

#### 🟡 警告告警（心跳时汇报）
- 后台服务异常退出
- 发布延迟（scheduled 时间过后 5-10 分钟）
- cron 单次失败
- 选题库存不足（<3 个可用选题）

### 正常响应

无异常时回复：`HEARTBEAT_OK ✅`

有异常时回复：具体告警内容

## 发布闭环检查规则

**检查时机**: 每个整点过 10 分（XX:10）  
**检查逻辑**:
1. 读取 `latest-article.json` 的 `scheduledPublishAt`
2. 如果当前时间 > scheduledPublishAt + 10 分钟
3. 检查是否有 `publishedAt` 或 `status: "published"`
4. 无 → 触发告警，尝试重发或通知人工介入
