BMW 沙盘系统
把一个 .xlsm 文件变成 Web 应用

原来 vs 现在

原来
  • 一个 .xlsm 文件，44 个 Sheet，1359 个公式，678 个跨表引用
  • 25 个循环引用，靠 Excel 迭代计算
  • 只能一个人打开用，数据在本地，无法共享
  • 改一个数字要等重算，改多了就卡死
  • 没有版本管理，改了不知道谁改的
现在
  • 44 个 HTML 页面，从源文件解析，不硬编码任何数据
  • 彩色 S 曲线 + 箭头，从单元格边缘出发（V28 迭代）
  • 构建完整依赖图，识别所有循环引用
  • 35 家经销商 KPI 数据库，随时可查
  • 计算引擎 14 项指标 100% 对齐 Excel

项目故事

📌 为什么做这个项目？

宝马的经销商沙盘是一个 Excel 文件，但它是整个售后业务的核心工具。教练用它做推演，经销商用它看差距。但这个文件太复杂了——44 个 Sheet，1359 个公式，25 个循环引用。每次打开都要等很久，而且只能一个人用。

🎯 目标：让 44 个 Sheet 变成 44 个网页，任何人都能随时查看

⚡ 最难的是什么？

不是把 Excel 转成 HTML——那个很简单。最难的是保留计算逻辑。1359 个公式里有 678 个跨 Sheet 引用，还有 25 个循环引用。Excel 用迭代计算处理循环引用，JS 需要实现同样的逻辑。

💡 关键教训：从源文件解析，不要硬编码。Codex 比手写代码质量好 3 倍。

🚀 结果

从 V1 到 V28，迭代了 28 个版本。最终 44 个 Sheet 全部变成 HTML，S 曲线从单元格边缘出发（而不是居中），计算引擎 14 项指标 100% 对齐。经验文档 296 行，以后遇到 Excel 复刻可以直接复用。

✅ 44 页面 · 1359 公式 · 678 跨表引用 · 14 项指标 100% 对齐

技术栈

Python / openpyxl
SVG 矢量图形
JavaScript 计算引擎

依赖图算法

AI 经销商诊断系统
10 分钟完成过去 2 天的工作

原来 vs 现在

原来
  • 教练进店辅导：收集 Excel → 手动算 5 大指标 → 对比历史 → 手写报告
  • 一家店至少 2 天，35 家店需要几个月
  • 报告质量完全依赖教练个人经验
  • 不同教练出的诊断标准不统一
  • 没有行业基准，不知道"好"是什么水平
现在
  • 数据喂给 AI：自动提取 KPI → 对比行业基准 → 自动判断状态 → 自动生成报告
  • 10 分钟，35 家店的完整诊断报告
  • 诊断逻辑标准化，新人也能出高质量报告
  • 34 家店的均值/中位数/分位数作为行业基准
  • 每家店自动判定：优秀 / 正常 / 预警 / 异常

项目故事

📌 为什么做这个项目？

辅导了 35 家奔驰店，手里攒了近 100 份真实数据。每次教练进店，需要收集经销商 Excel（销售、库存、客流、转化…），手动计算 5 大核心指标，对比历史数据找差距，手写诊断报告和建议。一家店至少 2 天。35 家店，几个月。

💡 洞察：AI 不会替代教练，但能把"算数据、找差距、写报告"这些耗时活儿全包了

⚡ 核心逻辑

把 35 家店的真实数据做成基准数据库。新来的经销商数据输入后，AI 自动提取 5 大指标（订单转化率、达成率、人效、库存占比、客流转化率），对比 34 家店的均值/中位数/分位数，自动判断状态，生成诊断报告 + 辅导建议。

🎯 5 大指标 · 34 家店基准 · 4 级状态判定 · 10 分钟出报告

🚀 结果

诊断报告从 2 天缩短到 10 分钟，效率提升 12 倍。35 家奔驰门店全部部署，诊断逻辑标准化。经验文档沉淀为可复用的方法论。

✅ 35 家店 · 100+ 份报告 · 12× 效率提升

AI 企微运营引擎
1000 客户 × 1000 套话术

原来 vs 现在

原来
  • 群发模板消息，千人一面，客户反感
  • 销售离职带走客户，企业资产流失
  • 不知道客户什么时候有需求，被动等待
  • 客户信息散落在销售个人手机里
  • 只能上班时间回复，非工作时间失联
现在
  • 工单 + 销售记录 + 个人信息 + 外网数据 → 完整客户画像
  • 1000 客户 × 1000 套话术，千人千面
  • Agent 判断时机：该保养了、该换车了 → 主动触达
  • 所有客户数据沉淀在企业系统里，不依赖个人
  • 24 小时在线，秒回，不流失任何机会

项目故事

📌 为什么做这个项目？

去年试过微信机器人，后面接大模型训好固定话题，主动加人聊天。逻辑上 OK，但平台封锁太严。现在换了思路——不做"破解平台"的事，而是帮客户把"自己的平台"AI 化。企微有官方 API，数据在经销商自己手里，完全合规。

💡 核心理念：不是"群发广告"，而是基于画像的精准沟通

⚡ 核心能力

客户进群 → Agent 自动欢迎 + 建档。客户发消息 → Agent 理解意图 → 回复或转人工。客户不说话 → Agent 定期触达（保养提醒、活动邀请、新车资讯）。后台持续学习 → 每次对话更新画像 → 下次更精准。

🎯 千人千面 · 24h 在线 · 画像驱动 · 持续学习

🚀 商业价值

一套系统，可以卖给 N 个经销商。边际成本趋近于 0。每个经销商/4S 店都是客户，按客户数收费。保险公司、二手车商、银行车贷部门都可以接。

📈 1000 客户 · 1000 套话术 · SaaS 模式 · 边际成本趋 0
