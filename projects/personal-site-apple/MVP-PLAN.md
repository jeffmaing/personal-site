# 数字麻明 — MVP 构建计划

> 概念：整个网站 = 数字麻明，不是简历+聊天框

## 零低级错误清单（Codex 写代码前必须确认）

### 🚫 设计层面
1. **绝不出现 emoji** — 所有装饰用 SVG 几何图形或纯文字
2. **字体系统** — 只用系统字体栈，不加载外部字体（避免阻塞渲染）
3. **颜色不超过 5 种** — 主色 #1d1d1f，辅色 #86868b，强调色 #0071e3，背景 #fff/#f5f5f7，分隔线 #e8e8ed
4. **响应式** — 手机优先设计，最小宽度 375px 必须完美
5. **字号阶梯** — 标题 36/28/22/18/15/13，不用随意数字
6. **间距系统** — 4 的倍数：4/8/12/16/24/32/48/64

### 🚫 代码层面
7. **TypeScript 严格模式** — 不用 any，不用 @ts-ignore
8. **组件文件结构** — 每个组件一个文件，不写超 200 行的组件
9. **状态管理** — 用 React useState，不用复杂状态库
10. **动画** — 只用 CSS transition/animation，不用 JS 动画库
11. **事件监听** — scroll 用 passive: true，用完 removeEventListener
12. **IntersectionObserver** — 用完 disconnect，避免内存泄漏
13. **key prop** — 列表用唯一 key，不用 index
14. **图片懒加载** — 用 loading="lazy"，首屏图片预加载
15. **构建检查** — 部署前必须 `npm run build` 通过

### 🚫 用户体验
16. **首屏加载 < 2 秒** — 总 JS 体积 < 150KB gzip
17. **无阻塞资源** — 所有 CSS inline 或首屏内联
18. **触摸反馈** — 点击有 active 状态，不能无响应
19. **滚动流畅** — 不能有布局抖动（CLS = 0）
20. **键盘可用** — Tab 能遍历所有可交互元素

## Codex 协作流程

### 我的任务
1. 写好 site-config.ts（所有数据）
2. 写好组件接口定义（Props types）
3. 设计好 CSS 变量和样式系统
4. 验收每个组件的渲染效果

1. 实现组件 JSX
2. 实现交互逻辑（状态切换、事件处理）
3. 实现 CSS 样式（遵循设计规范）
4. 通过 TypeScript 编译检查
